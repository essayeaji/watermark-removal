# DMIT云服务器教程：如何配置自定义SSH密钥替代默认密钥

## 为什么需要自定义SSH密钥？

DMIT云服务器默认采用密钥登录机制，创建VPS时会自动生成SSH密钥对。但需要注意两个关键点：

1. **密钥只能下载一次** - 首次登录时会弹出下载提示（也可通过右上角"SSH密钥管理"获取）
2. **默认密钥不便管理** - 建议使用"下载私密密钥"按钮获取密钥文件（公开密钥通常无需下载）

## 详细配置指南

### 1. 准备工作
确保已准备好：
- Xshell或其他SSH客户端工具
- 可用的公钥/私钥对（可通过ssh-keygen生成）

### 2. 密钥上传步骤

#### 方法一：使用Xshell生成密钥
1. 在Xshell中创建新密钥对
2. 导出公钥文件(.pub格式)
3. 登录DMIT控制面板

#### 方法二：手动配置流程
1. 进入VPS管理面板
2. 找到"SSH Public Key"选项
3. 点击编辑，粘贴你的公钥内容
4. 保存设置

### 3. 生效验证
完成配置后需要：
1. 重启服务器使配置生效
2. 使用新密钥测试连接

## 两种登录方式对比

| 登录方式 | 优点 | 缺点 |
|---------|------|------|
| 默认密钥 | 自动生成，开箱即用 | 密钥仅能下载一次，不便管理 |
| 自定义密钥 | 可复用现有密钥，便于统一管理 | 需要额外配置步骤 |

## 最佳实践建议

1. 建议同时保留默认密钥作为备用
2. 定期轮换密钥提高安全性
3. 为不同服务器使用不同密钥对

> 提示：DMIT云服务器采用密钥认证机制，相比密码登录更安全可靠。合理配置SSH密钥能显著提升服务器管理效率。
